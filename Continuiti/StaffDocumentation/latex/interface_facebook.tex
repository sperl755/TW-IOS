\-Starts a dialog which prompts the user to log in to \hyperlink{interface_facebook}{\-Facebook} and grant the requested permissions to the application.

\-If the device supports multitasking, we use fast app switching to show the dialog in the \hyperlink{interface_facebook}{\-Facebook} app or, if the \hyperlink{interface_facebook}{\-Facebook} app isn't installed, in \-Safari (this enables single sign-\/on by allowing multiple apps on the device to share the same user session). \-When the user grants or denies the permissions, the app that showed the dialog (the \hyperlink{interface_facebook}{\-Facebook} app or \-Safari) redirects back to the calling application, passing in the \-U\-R\-L the access token and/or any other parameters the \hyperlink{interface_facebook}{\-Facebook} backend includes in the result (such as an error code if an error occurs).

\-See \href{http://developers.facebook.com/docs/authentication/}{\tt http\-://developers.\-facebook.\-com/docs/authentication/} for more details.

\-Also note that requests may be made to the \-A\-P\-I without calling authorize() first, in which case only public information is returned.
